## Описание

Необходимо реализовать backend-поддержку настройки **количества доступных дней отпуска** для каждого типа отпуска.

Эта настройка определяет максимальное количество дней определённого типа отпуска, которое сотрудник может использовать **в течение одного календарного года**.

Настройка задаётся администратором в настройках типа отпуска.

В рамках текущей реализации поддерживается только один вариант:

**фиксированное количество дней в году.**

В будущем планируется добавить дополнительные параметры (например, возможность брать дни в минус), но они **не входят в текущую реализацию**.

# Поведение настройки

Для каждого типа отпуска администратор может задать:

* фиксированное количество дней отпуска в году.

Например:

| Тип отпуска | Доступные дни |
| --- | --- |
| Основной отпуск | 30 |

Это означает, что сотрудник может использовать **не более 30 дней этого типа отпуска в течение одного календарного года**.

---

# Логика проверки при создании заявления

При подаче заявления на отпуск система должна выполнять проверку доступного количества дней.

Проверка выполняется для:

* подтверждённых отпусков;
* ожидающих подтверждения отпусков.

Это необходимо для предотвращения подачи заявок, превышающих установленный лимит.

### Пример

Если установлен лимит **30 дней в году**:

1. Сотрудник подал заявку на **20 дней** (статус: ожидает подтверждения).
2. Система считает эти дни уже использованными.
3. Если сотрудник пытается подать ещё одну заявку на **20 дней**, система должна отклонить такую заявку.

Причина:
общая сумма дней (20 + 20) превышает лимит **30 дней в году**.

---

# Правила расчёта

При проверке лимита система должна учитывать:

* все подтверждённые отпуска данного типа;
* все отпуска, находящиеся в статусе ожидания подтверждения;
* только отпуска текущего календарного года.

Если суммарное количество дней превышает установленный лимит, подача заявки должна быть запрещена.

---

# Валидации

Backend должен выполнять следующие проверки:

* значение количества дней должно быть положительным числом;
* значение не может быть равно нулю;
* при создании заявки проверяется суммарное количество дней;
* если лимит превышен, заявка не может быть создана.

---

# Ограничения текущей реализации

В рамках данной задачи:

* поддерживается только **фиксированное количество дней в году**;
* сотрудник **не может брать дни сверх установленного лимита**;
* возможность брать отпуск **в минус** будет реализована позже.
