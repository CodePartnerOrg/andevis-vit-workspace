# BE: Раздел Локализация
/
В системе уже существует механизм хранения переводов значений. Однако в данный момент переводы могут добавляться или изменяться только на уровне разработки.

Типы документов и уровни образования используются в профиле работника и поступают из внешней системы **Virosoft**. Каждая запись содержит код-идентификатор, который используется для интеграции между системами.

Этот код является неизменяемым и должен сохраняться при всех операциях.

В настоящее время администраторам системы недоступна возможность изменять переводы этих значений через интерфейс. Это усложняет поддержку локализации и требует участия разработчиков для любых изменений отображаемых названий.

---

# Business Value

Добавление backend поддержки редактирования переводов позволит:

* администраторам самостоятельно управлять локализацией справочников
* сократить зависимость от разработчиков
* быстрее корректировать отображаемые названия
* обеспечить единообразное отображение справочников во всей системе

---

# Описание задачи

Необходимо реализовать backend поддержку административного раздела **Локализация**, который позволит редактировать переводы значений справочников.

Название раздела:

| ET | EN | RU |
| --- | --- | --- |
| Lokaliseerimine | Localization | Локализация |

На первом этапе раздел должен поддерживать два справочника, используемых в профиле работника:

* Типы документов
* Уровни образования

---

# Работа со справочниками

Каждая запись справочника содержит:

* **код-идентификатор из Virosoft**
* **оригинальное название на эстонском языке**, полученное из Virosoft
* **локализованные названия**, используемые в системе

Особенности:

* код-идентификатор является **неизменяемым**
* используется системой для интеграции
* эстонское значение по умолчанию — это значение, полученное из Virosoft

---

# Редактируемые данные

Администратор может изменять только локализованные названия.

Поддерживаемые языки:

* ET
* EN
* RU

Ограничения:

* код-идентификатор нельзя изменять
* изменения применяются только внутри текущей системы
* данные не отправляются обратно в Virosoft

---

# Логика локализации

Во всех местах системы, где используются значения справочников:

* значения должны отображаться на языке интерфейса пользователя

Если перевод на выбранный язык отсутствует, используется следующий fallback:

1. язык пользователя
2. эстонский язык

# Acceptance Criteria

* [ ] Backend использует существующий механизм хранения переводов
* [ ] Реализована возможность обновлять переводы значений локализации
* [ ] Поддерживаются справочники:
    * [ ] Типы документов
    * [ ] Уровни образования
* [ ] Каждая запись содержит код-идентификатор из Virosoft
* [ ] Код-идентификатор **нельзя изменять**
* [ ] Поддерживаются переводы:
    * [ ] ET
    * [ ] EN
    * [ ] RU
* [ ] Эстонский язык является обязательным
* [ ] Изменения сохраняются только в текущей системе
* [ ] При сохранении данные **не отправляются в Virosoft**
* [ ] При синхронизации из Virosoft существующие переводы **не перезаписываются**
* [ ] Backend возвращает значения согласно языку интерфейса пользователя
* [ ] Реализован fallback переводов:
    * [ ] язык пользователя → ET
* [ ] Отредактированные значения используются в профиле работника
* [ ] Архитектура решения позволяет добавлять новые справочники в будущем
