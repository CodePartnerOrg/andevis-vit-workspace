# BE: Общие настройки отпусков для администратора

## Описание

Необходимо реализовать backend-поддержку для страницы **общих настроек отпусков**
**Puhkuste üldised seadistused / General vacation settings / Общие настройки отпусков**.

Backend должен:

* хранить значения общих настроек отпусков;
* возвращать их для отображения на фронтенде;
* сохранять изменения настроек;
* поддерживать как текущие, так и будущие настройки.

Настройки применяются **на уровне организации**.

Также backend должен предоставлять список типов отпусков, чтобы фронтенд мог отображать таблицу **Puhkuste seaded** (настройки типов отпусков).

---

# Общие настройки отпусков

Общие настройки представляют собой набор параметров, которые управляют поведением отпускного модуля в системе.

Каждая настройка имеет тип **включено / выключено**.

Настройки сохраняются отдельно для каждой организации.

---

# Настройки, которые должны поддерживаться

Backend должен поддерживать хранение и изменение следующих настроек:

| Настройка |
| --- |
| Puhkusegraafik on kõigile töötajatele kättesaadav |
| Kolleegide tegelike puhkuste vaatamine |
| Puhkuse muutmine pärast kinnitamist |
| Mitte kuvada tagasi lükatud puhkusi töötaja kalendris |
| Alustamata puhkuse tühistamine pärast kinnitamist |
| Puhkuse tühistamine ilma kinnitamiseta |
| Puhkusejääk sealhulgas ootel olevad puhkused |
| Piira puhkuse esitamist / muutmist / tühistamist, kui soovitakse puhkuse raha ette saada |

Backend должен обеспечивать:

* получение текущих значений настроек;
* сохранение изменений;
* использование значений по умолчанию, если настройка ранее не задавалась.

---

# Будущие настройки

Следующие настройки планируются к реализации позже, но backend должен быть подготовлен для их хранения.

| Настройка | Статус |
| --- | --- |
| Muudatuste sünkroonimine pärast kinnitamist | будущая реализация |
| Aruane puhkuseavalduste kohta | будущая реализация |
| Kinnitatud puhkuse kustutamine | on hold |

Для этих настроек backend должен:

* поддерживать хранение значения;
* возвращать значение в настройках.

Бизнес-логика, связанная с этими настройками, будет реализована в отдельных задачах.

---

# Список типов отпусков

Backend должен возвращать список всех типов отпусков, доступных в системе.

Для каждого типа отпуска должны передаваться следующие данные:

* код типа отпуска;
* название типа отпуска;
* круг подтверждения (approval workflow);
* статус типа отпуска.

Этот список используется для отображения таблицы **Puhkuste seaded** на странице настроек.

---

# Связь с детальными настройками отпусков

Каждый тип отпуска имеет детальные настройки.

Переход к редактированию типа отпуска выполняется через отдельную страницу.

Backend должен обеспечивать возможность получения данных конкретного типа отпуска для последующего редактирования.

Детальные настройки типов отпусков реализуются **в отдельной задаче**.

---

# Правила

* Настройки применяются на уровне организации.
* Если настройка не была изменена организацией, используется значение по умолчанию.
* Изменение настроек одной организации не влияет на другие организации.
* Настройки должны сохраняться и корректно возвращаться при последующих запросах.

# Технические детали

В этой задаче необходимо отойти от текущей модели складывать каждое поле настроек в отдельную запись в таблице настроек.
Т.к. новая реализация на фронтенде уже подразумевает отдельную большую секцию общих настроек для отпусков, 
а также настройки по каждому типу отпусков, необходимо реализовать текущую задачу с учетом последующих сохранение настроек по конкретному типу отпусков.
Структура АПИ и Сваггер должны отражать это. Для общих настроек хорошо бы работать с DTO в котором будет все настройки.
Возможно в дальнейшем этот подход переедет на подобную реализацию других секций настроек. Все эти настройки в будущем
будут использоваться на бэкэнде для валидации и расчетов с ними должно быть удобно работать.
Новый функционал на бэкэнде необходимо покрыть тестами
