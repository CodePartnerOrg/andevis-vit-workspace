# Кандидаты: Автоматическое удаление контейнера договора в Dokobit после завершения подписания

Необходимо реализовать механизм автоматического удаления контейнера договора во внешнем сервисе Dokobit после успешного завершения процесса подписания всеми сторонами.

Текущая реализация сохраняет контейнеры в Dokobit после подписания, что приводит к накоплению данных во внешней системе и потенциальным рискам по хранению избыточной информации.

---

**Requirements:**

1. **Триггер удаления:**

    * Удаление должно выполняться только после получения подтверждения о завершении подписания (completed / signed).
    * Источник события:
        * webhook от Dokobit **или**
        * внутренняя логика (после финализации статуса договора).

2. **Интеграция с Dokobit API:**

    * Использовать соответствующий endpoint для удаления контейнера.
    * Обеспечить корректную аутентификацию и обработку ошибок API.

3. **Идемпотентность:**

    * Повторные попытки удаления не должны приводить к ошибкам (обработка 404 / already deleted).

4. **Обработка ошибок:**

    * Логирование всех неуспешных попыток удаления.

5. **Хранение идентификаторов:**

    * Убедиться, что containerId (или аналог) сохраняется и доступен для удаления.
    * В случае отсутствия — предусмотреть fallback/логирование.

6. **Транзакционность:**

    * Удаление контейнера не должно ломать основной бизнес-процесс (договор считается подписанным независимо от результата удаления).
    * Выполнять асинхронно.

7. **Безопасность и аудит:**

    * Логировать факт удаления (containerId, timestamp, результат).
    * Не хранить чувствительные данные из контейнера после удаления.

---

**Acceptance Criteria:**

* [ ]  После перехода договора в статус "подписан".
* [ ]  Контейнер в Dokobit удаляется автоматически.
* [ ]  Повторный вызов удаления не приводит к падению.
* [ ]  Основной процесс подписания не зависит от удаления.
